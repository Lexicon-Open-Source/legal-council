## Design Context

### Users

The council app is for legal practitioners, researchers, and operators who need to turn a case summary into a structured deliberation and draft legal opinion. They use it as a focused working surface: input facts, review three judicial perspectives, ask follow-up questions, and export an opinion without losing trust in the reasoning trail.

### Brand Personality

Calm, analytical, approachable. The interface should feel like a careful deliberation room inside the Lexicon platform, not a dramatic courtroom simulation or a generic AI chat product. It should communicate restraint, clarity, and legal seriousness while staying easy to scan.

### Aesthetic Direction

Use Lexicon's design-system color system. The canonical platform reference is `lexicon-procurement`; `admin-dashboard` is the reference for app-shell token discipline. This app ships light mode only for this revamp. Headings use Rethink Sans; body and UI text use the design system's system-font stack.

The three judge identities are functional signals and must remain paired with iconography: Legalis uses purple, Humanis uses green, and Sejarawan uses blue. Brand teal is reserved for primary actions and single-context emphasis. Avoid decorative AI-dashboard tropes, ornamental gradients, flag-color motifs, batik motifs, glow effects, and color-only identity.

### Design Principles

1. Keep deliberation readable before making it decorative.
2. Use shared design-system primitives and tokens as the default building blocks.
3. Preserve the linear flow: input perkara, musyawarah, pendapat hukum.
4. Use teal sparingly so primary actions stay obvious.
5. Treat agent color as identity, not status or judgment.
